前言：

当数据量大的时候，一台机器可能不足以满足性能的需求，如存储空间、内存不足，而垂直扩展十分昂贵，故用分片进行水平扩展是一个很好的解决方法。

本文介绍了mongodb 进行分片的方法和步骤

mongodb分片 结构

主要由下面的三部分组成

  • Shard:

    • 可以使mongod也可以是几台机器组个一个replica set，防止主机单点故障

  • Config Server:

    • 存储整个集群的配置信息，如 chunk 大小。

  • Query Routers:

    • mongos实例，是客户端的入口。让整个集群看上去像单一数据库，前端应用可以透明使用。
